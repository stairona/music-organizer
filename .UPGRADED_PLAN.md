## 🏗️ Proposed Architecture

```
┌─────────────────┐    HTTP/REST    ┌─────────────────┐
│  Desktop App    │◄──────────────►│   FastAPI       │
│  (Electron)     │                 │   Backend       │
└─────────────────┘                 └─────────────────┘
                                          │
                                    ┌─────┴─────┐
                                    ▼           ▼
                            Spotify Web   spotdl CLI
                              API          (subprocess)
                                    │           │
                                    └─────┬─────┘
                                          ▼
                            ┌─────────────────────────┐
                            │  Organized Library      │
                            │  (via existing logic)   │
                            └─────────────────────────┘
```

**Key invariants**:
- Backend remains single-author local server (no multi-user sessions)
- spotdl runs as subprocess with stdout parsing for progress
- OAuth tokens stored encrypted in `~/.config/music-organizer/` (sqlite or JSON)
- Download tasks persisted with state machine: `queued` → `downloading` → `completed`/`failed`
- Progress snapshots stored as JSON blobs (for resumable UI polling)
- Desktop app uses polling (`GET /downloads/{id}/status`) every 2-3s

- Store new tables in separate SQLite file (e.g., `~/.config/music-organizer/spotify.db`) to avoid migrating existing `store/__init__.py` schema; if using same store, ensure backward-compatible `CREATE TABLE IF NOT EXISTS`.
- Never overwrite user's existing `config.json`; extend it with new `spotify` section if missing.

## ✅ Decision Log (from prior context)

| Decision | Rationale |
|----------|-----------|
| Desktop-first (not web) | Easier local file access, no deploy, simple auth |
| FastAPI remains local HTTP server | Desktop app can call as localhost; reuse existing service layer |
| Polling (not websockets) | Simpler, sufficient for 1-5 active downloads |
| spotdl as subprocess (not library) | spotdl primarily CLI; avoids dependency hell |
| OAuth PKCE (no client secret) | Desktop apps can't hide secrets; PKCE is recommended |
| auto_organize by default | Seamless UX: download → organize → ready to DJ |
| Separate Spotify store tables | Avoid complicating existing run history; clear boundary |

## ⚠️ Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| spotdl API/CLI changes break parsing | High | Pin spotdl version in requirements; implement feature flag to disable spotdl if broken |
| Spotify token refresh fails silently | Medium | Log errors; show "Reconnect Spotify" in desktop UI; keep old tokens for read-only until expiry |
| Desktop app startup race (backend not ready) | Low | Add health check endpoint `GET /health`; desktop retries with backoff |
| Download fills disk | High | Pre-check free space vs estimated size; stop if threshold (<20%) |
| Cross-platform path issues (Windows later) | Medium | Use `pathlib` everywhere; test on macOS only for now |
