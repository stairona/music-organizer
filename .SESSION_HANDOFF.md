### v2.1.0 Phase 2 — Spotify Authentication Service (OAuth 2.0 PKCE) (2026-03-16)
**Commit**: `9b713bf` — "feat: implement Spotify OAuth PKCE authentication (Phase 2)"
**Status**: ✅ Complete and pushed to `origin/main`

#### New Service Module
`app/backend/services/auth_service.py`:
- `generate_pkce_pair()`: create URL-safe verifier + SHA256 challenge
- `get_auth_url(verifier, state)`: build Spotify authorization URL (URL-encoded)
- `exchange_code_for_tokens(code, verifier)`: POST to Spotify token endpoint
- `refresh_access_token(refresh_token)`: handle token renewal (preserves refresh if not returned)
- `load_oauth()`, `store_oauth(tokens)`, `logout()`
- `is_token_expired(ts, buffer)`: expiry check with safety buffer
- `get_valid_access_token()`: auto-refresh convenience for API calls

#### New API Routes
- `GET /api/v1/auth/spotify/login` → returns `{auth_url, state, code_verifier}` (desktop app opens URL in browser)
- `POST /api/v1/auth/spotify/callback` → body: `{code, code_verifier}` → exchanges for tokens
- `GET /api/v1/auth/spotify/status` → returns `{connected: bool}` (auto-refresh if token expired)
